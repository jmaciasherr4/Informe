# EQUIPO F: Software Ideas Modeler

## Alcívar

**Teoría:**

- Software Ideas Modeler es una herramienta versátil para el modelado UML y la generación de código.
- Los diagramas creados pueden utilizarse tanto como documentación como base para el desarrollo del software.
- Permite elaborar diagramas de requisitos siguiendo el estándar UML.
- Facilita la organización de los requisitos mediante atributos como nombre, ID, prioridad y descripción.

**Práctica:**

Mostró la interfaz principal de la herramienta y creó un diagrama de requisitos basado en el estándar UML. Agregó un requisito con su nombre, ID, prioridad y descripción, y posteriormente incorporó dos requisitos adicionales hasta completar un total de tres.

---

## Quintero

**Teoría:**

- Permite acelerar la construcción inicial del backend y la estructura de clases.
- Mantiene la trazabilidad entre requisitos y modelos del sistema.
- Facilita la evolución del software mediante ingeniería directa e inversa.
- Como limitación, la generación automática no cubre lógica de negocio ni validaciones complejas.

**Práctica:**

Presentó el caso de estudio AquaGest y explicó el uso de la herramienta para cumplir restricciones del proyecto. Mostró cómo crear una clase llamada **Estanque**, agregó los atributos **idEstanque** y **nombreEstanque**, además de la operación **registroEstanque()**. Generó el código en Visual Studio C#, realizó ingeniería inversa y verificó la validación estructural del modelo.

## Barrionuevo

**Teoría:**

- Permite generar código automáticamente a partir de diagramas UML.
- Soporta estereotipos y perfiles UML para personalizar los modelos.
- Incluye herramientas de validación de sintaxis y semántica antes de generar código.
- Conserva bloques protegidos y ofrece una interfaz ligera e intuitiva.

**Práctica:**

Mostró la creación de un diagrama de casos de uso para un módulo del sistema. Agregó los actores **Administrador**, **Operador de Productos** y **Técnico Acuícola**, incorporó seis casos de uso y estableció las relaciones correspondientes entre actores y casos de uso. También explicó el uso de la relación **include**, aunque señaló que fue aplicada incorrectamente en un caso.
